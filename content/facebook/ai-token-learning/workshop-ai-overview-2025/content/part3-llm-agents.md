### 1.1. LLM là gì?

> **LLM (Large Language Model)** = Model được train trên TOÀN BỘ văn bản internet.
> 1 model biết "gần như mọi thứ": viết code, dịch thuật, phân tích pháp lý, trả lời y khoa...

**Analogi:**

> LLM giống như một người đã đọc hết thư viện quốc gia — và có thể **TRẢ LỜI** bất kỳ câu hỏi nào dựa trên kiến thức đó.
>
> Không cần hỏi "Google cách làm X" nữa. Hỏi LLM.

### 2.1. Tại Sao Agentic AI Là Bước Nhảy Lớn?

**Sự khác biệt cốt lõi:**

| | LLM (Chatbot) | Agentic AI (Autonomous) |
|---|---|---|
| **Bạn nói** | "Dịch đoạn này sang tiếng Anh" | "Book cho tôi một chuyến bay HCM → HN ngày mai" |
| **AI làm** | [Dịch] | 1. Tìm chuyến bay → 2. So sánh giá → 3. Book + thanh toán → 4. Gửi confirmation email |
| **Kết quả** | Chỉ TRẢ LỜI | TỰ HÀNH ĐỘNG → KẾT QUẢ |

> **Nói thẳng:** LLM là **thư ký** — trả lời câu hỏi. Agentic AI là **nhân viên** — hoàn thành công việc.

---

### 2.2. Kiến Trúc Agent — Ba Bước Tư Duy

Agentic AI hoạt động theo 3 bước, lặp lại cho đến khi đạt mục tiêu:

**Bước 1 — PLAN (Lập kế hoạch)**

- "Mục tiêu là gì? Cần làm những bước nào?"
- Break down task thành sub-tasks
- Prioritize thứ tự

**Bước 2 — ACT (Hành động)**

- "Thực hiện bước tiếp theo"
- Call tools (search, code, API, file system)
- Gọi LLM để suy luận

**Bước 3 — REFLECT (Suy ngẫm)**

- "Kết quả đúng chưa? Cần điều chỉnh gì?"
- Evaluate output
- Retry nếu sai → next step nếu đúng

→ **Lặp lại cho đến khi GOAL REACHED**

---

**Ví dụ thực tế — "Research Agent":**

> User: "Research competitors của startup X trong 5 phút"
>
> **Plan:** Tìm 5 competitors → Research mỗi cái → Tổng hợp report
> **Act:** Search web → Đọc website → Đọc báo cáo tài chính
> **Reflect:** Đánh giá chất lượng → Bổ sung nếu thiếu → Viết report
> **Output:** Report 5 trang về competitors

### 2.3. MCP + A2A Protocol — Tiêu Chuẩn 2026 🔬VERIFY

> 🔬 Từ NotebookLM Research: Năm 2026, hai protocol đang định hình Agentic AI ecosystem.

**Tại sao quan trọng?**

> "Trước đây, mỗi AI tool có cách kết nối riêng. Giống như ổ cắm điện Nhật/Mỹ/Châu Âu — mỗi loại khác nhau.
> MCP/A2A giống như **USB-C** — một chuẩn kết nối CHUNG cho tất cả AI."

| Protocol | Ai phát triển | Mục đích | Ví dụ |
|---|---|---|---|
| **MCP** (Model Context Protocol) | Anthropic | AI Model ↔ External Tools / Data Sources | Claude kết nối với Google Drive, Slack, GitHub, Database |
| **A2A** (Agent-to-Agent) | Google + Multi-vendor | Agent A ↔ Agent B (giao tiếp với nhau) | Research Agent nói chuyện với Writer Agent → Output hoàn chỉnh |

> "Hiểu MCP/A2A = biết tương lai của AI Agent. Nếu bạn build AI product, dùng MCP/A2A = product của bạn **KẾT NỐI** với mọi thứ khác."

## MODULE 3 — MCP Deep Dive + Live Demo

### 3.1. MCP Là Gì — Ý Tưởng, Không Cần Code

MCP = Model Context Protocol — cách Claude/GPT kết nối với THẾ GIỚI BÊN NGOÀI.

```
Claude Agent ── MCP ──→ Google Search API
Claude Agent ── MCP ──→ Wikipedia
Claude Agent ── MCP ──→ Your Database
Claude Agent ── MCP ──→ Send Email
Claude Agent ── MCP ──→ Create Calendar Event
Claude Agent ── MCP ──→ Update CRM
```

> **Analogi:** "MCP giống như cho AI một bộ CÔNG CỤ. Không có công cụ, AI chỉ nói chuyện. Có công cụ, AI có thể **LÀM** việc."
