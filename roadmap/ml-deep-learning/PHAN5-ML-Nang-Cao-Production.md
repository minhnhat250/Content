## 5.1 — Time Series Forecasting

### Mục đích
Time Series = dữ liệu có thứ tự thời gian (giá cổ phiếu, doanh số bán hàng, thời tiết). Khác với data thường: quan hệ phụ thuộc vào thời gian, có seasonality, trend, noise. Bài toán: dự đoán giá trị tương lai dựa trên quá khứ.

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Time Series — khác gì data thường** | [Scaler — Time Series](https://www.scaler.com/topics/time-series/time-series-analysis/) | Time series data có 3 thành phần: Trend (xu hướng dài hạn), Seasonality (chu kỳ lặp lại: theo tháng, theo quý), Noise (nhiễu ngẫu nhiên). Cross-sectional data: quan sát tại 1 thời điểm, nhiều individuals. Panel data: kết hợp cả hai. Time series cần xử lý temporal dependencies — observation tại t phụ thuộc t-1, t-2... |
| **Stationarity & ADF test** | [Scaler — Stationarity](https://www.scaler.com/topics/time-series/stationarity-in-time-series/) | Stationary = statistical properties (mean, variance) không đổi theo thời gian. Non-stationary: trend, seasonality làm mean/variance thay đổi. ARIMA yêu cầu data phải stationary. ADF (Augmented Dickey-Fuller) test: kiểm định null hypothesis = "có unit root" (non-stationary). p-value < 0.05 → reject null → stationary. Nếu non-stationary: differencing (lấy difference giữa t và t-1) → làm stationary |
| **Decomposition — trend, seasonality, residual** | [Scaler — Decomposition](https://www.scaler.com/topics/time-series/time-series-decomposition/) | Phân tách chuỗi thành 3 thành phần: Trend (T): xu hướng dài hạn. Seasonal (S): dao động chu kỳ. Residual (R): phần còn lại. Additive: Y = T + S + R (dùng khi seasonality constant). Multiplicative: Y = T × S × R (dùng khi amplitude tăng theo trend). Decomposition giúp hiểu cấu trúc data trước khi model |
| **Moving Average & Exponential Smoothing** | [Scaler — Moving Avg](https://www.scaler.com/topics/time-series/moving-average-in-time-series/) | Moving Average (MA): trung bình của K điểm gần nhất. Smooths out noise để thấy trend rõ hơn. Simple Moving Average: tất cả K điểm weight bằng nhau. Weighted MA: điểm gần hơn weight cao hơn. Exponential Smoothing: EWMA (Exponentially Weighted MA). α = smoothing factor (0-1). α nhỏ → smooth nhiều (lag cao). α lớn → responsive (lag thấp). Dùng để smooth data hoặc làm baseline model |
| **ARIMA — AutoRegressive Integrated Moving Average** | [Scaler — ARIMA](https://www.scaler.com/topics/time-series/arima-model-in-time-series/) | ARIMA = AR (AutoRegressive) + I (Integrated = differencing) + MA (Moving Average). AR(p): dự đoán dựa trên p giá trị quá khứ. I(d): số lần differencing để stationary. MA(q): dự đoán dựa trên q sai số quá khứ. Notation: ARIMA(p, d, q). Ví dụ: ARIMA(2,1,2) = AR(2) + I(1) + MA(2). ACF/PACF plots dùng để chọn p, q. Auto-ARIMA: tự động tìm optimal p,d,q bằng AIC/BIC |
| **SARIMA — ARIMA với Seasonality** | [Scaler — SARIMA](https://www.scaler.com/topics/time-series/sarima-model/) | SARIMA = ARIMA + Seasonal component. Notation: SARIMA(p,d,q)(P,D,Q)[s]. (P,D,Q) = seasonal orders. [s] = seasonal period (s=12 cho monthly data, s=4 cho quarterly). Ví dụ: SARIMA(1,1,1)(1,1,1,12) cho monthly data với yearly seasonality. Cần thiết khi data có clear seasonal patterns (retail sales, temperature) |
| **Facebook Prophet — dự báo cho business** | [Prophet Official](https://www.scaler.com/topics/time-series/facebook-prophet/) | Prophet = time series forecasting tool của Facebook (Meta). Decomposition: y(t) = g(t) + s(t) + h(t) + ε(t). g(t) = trend (piecewise linear/logistic growth). s(t) = seasonality (Fourier series). h(t) = holidays effect. Ưu điểm: dễ dùng, tự xử lý missing values và outliers, có holidays effect, interpretable. Dùng khi: business data (sales, revenue), có seasonality rõ ràng, cần forecast nhanh. Code: `from prophet import Prophet` |
| **LSTM cho Time Series** | [Scaler — LSTM TS](https://www.scaler.com/topics/deep-learning/lstm-for-time-series/) | LSTM có thể học long-range dependencies trong time series. Input: sequences of past values (window size = lookback). Architecture: LSTM layers → Dense output layer. Stateful vs stateless: stateful giữ hidden state giữa batches. Multivariate LSTM: nhiều features đầu vào cùng lúc. Kết hợp với attention → Temporal Fusion Transformer (TFT) — state-of-the-art cho multivariate time series |
| **DLinear & NLinear — Linear approach** | [Scaler — DLinear](https://www.scaler.com/topics/time-series/dlinear-model/) | DLinear (Direct Linear): decomposition time series → trend + remainder → Linear layers riêng biệt → sum. NLinear (Normalized Linear): subtract latest value → Linear → add back. Bất ngờ: simple linear models beat complex LSTMs trên many time series benchmarks (2022-2023). Giải thích: time series thường có strong trend/seasonality → linear decomposition đủ tốt. Transformers (PatchTST, TiDE) tiếp tục cải thiện |
| **Evaluation Metrics — MAPE, SMAPE, RMSE** | [Scaler — TS Metrics](https://www.scaler.com/topics/time-series/evaluation-metrics-time-series/) | MAE (Mean Absolute Error): mean(|y_pred - y_actual|). Không penalize outliers nặng như MSE. RMSE (Root Mean Squared Error): √mean((y_pred - y_actual)²). Same unit như target. MAPE (Mean Absolute Percentage Error): mean(\|error/target\|) × 100. Interpretable as percentage — dễ hiểu cho business. Nhược: undefined when target = 0. SMAPE (Symmetric MAPE): symmetric, bounded 0-200%. Chọn metric phù hợp với business context |
| **Thực hành: dự đoán giá cổ phiếu** | [Kaggle Stock](https://www.kaggle.com/datasets/jacksoncrow/stock-market-dataset) | Dataset: historical stock prices (Open, High, Low, Close, Volume). Approach 1: ARIMA/SARIMA cho univariate (Close price). Approach 2: LSTM multivariate (OHLCV). Approach 3: Prophet cho có holidays effect. Split: time-based (train on older, test on newer). Metric: RMSE hoặc directional accuracy (% ngày đoán đúng chiều giá). Visualization: actual vs predicted over time |
| **Thực hành: dự đoán doanh số bán hàng** | [Kaggle Store Sales](https://www.kaggle.com/competitions/store-sales-time-series-forecasting) | Dataset: Store sales data với promotions, holidays, oil price (influencing factors). Feature engineering: lag features (sales của tuần trước), rolling mean, day-of-week effect. Prophet: dễ dàng thêm holidays effects. XGBoost với lag features: competitive với deep learning. Compare MAPE across models |

## 5.2 — Anomaly Detection (Phát Hiện Bất Thường)

### Mục đích
Anomaly = điểm dữ liệu không tuân theo pattern "bình thường". Ứng dụng: phát hiện gian lận thẻ tín dụng, detect equipment failure, network intrusion, medical anomalies. Khác Classification: không có labels cho "anomaly" (unsupervised) hoặc anomaly rất hiếm (imbalanced).

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Anomaly Detection — tổng quan** | [Scaler — Anomaly](https://www.scaler.com/topics/machine-learning/anomaly-detection-in-machine-learning/) | 3 loại: Point anomalies (điểm đơn lẻ bất thường), Contextual anomalies (bất thường trong context cụ thể: 50°C ở Hà Nội bất thường nhưng ở sa mạc thì không), Collective anomalies (chuỗi điểm bất thường). Point anomaly = điểm đơn lẻ far from others. Collective = pattern bất thường. Methods: statistical, proximity-based, reconstruction-based, prediction-based |
| **Statistical Methods — Z-score, IQR** | [Scaler — Z Score](https://www.scaler.com/topics/machine-learning/z-score-for-anomaly-detection/) | Z-score: z = (x - μ) / σ. Nếu \|z\| > 3 → anomaly (99.7% data trong ±3σ theo Normal distribution). Đơn giản, fast, nhưng giả định Normal distribution. IQR (Interquartile Range): Q1 = 25th percentile, Q3 = 75th percentile. IQR = Q3 - Q1. Lower bound = Q1 - 1.5×IQR. Upper bound = Q3 + 1.5×IQR. Outliers nằm ngoài bounds. Không giả định distribution |
| **Isolation Forest** | [Scaler — Isolation Forest](https://www.scaler.com/topics/machine-learning/isolation-forest-in-machine-learning/) | Ý tưởng: anomalies "dễ isolate" hơn normal points. Build random trees: tại mỗi node, random chọn feature + random split value. Anomalies có short path lengths (isolated nhanh), normals có long paths. Anomaly score = average path length. Random split → efficient, không cần distances. Parameters: `n_estimators` (số trees), `contamination` (% anomalies expected). `sklearn.ensemble.IsolationForest` |
| **One-Class SVM** | [Scaler — OneClass SVM](https://www.scaler.com/topics/machine-learning/one-class-svm/) | Train chỉ trên "normal" data. Tìm boundary (hyperplane) enclosing normal points với maximal margin. Points nằm ngoài boundary = anomalies. Kernel: RBF, linear, polynomial. Phù hợp khi chỉ có normal data để train (không có anomaly examples). Nhược: không scale tốt cho large datasets (O(n²) training complexity) |
| **Autoencoder cho Anátom Detection** | [Scaler — Autoencoder](https://www.scaler.com/topics/deep-learning/autoencoder-for-anomaly-detection/) | Autoencoder = encoder (compress) + decoder (reconstruct). Train chỉ trên normal data. Normal points: encoder → latent → decoder → reconstruction gần với original. Anomalies: reconstruction error cao vì không fit vào normal patterns. Reconstruction error = anomaly score. Latent dimension < input dimension (bottle-neck) → buộc phải học essential features. Phù hợp cho high-dimensional data (images, sensor data) |
| **LSTM Autoencoder cho multivariate TS** | [Scaler — LSTM Autoencoder](https://www.scaler.com/topics/deep-learning/lstm-autoencoder-for-time-series-anomaly-detection/) | Dùng LSTM thay vì feed-forward layers để encode/decode sequences. Input: sliding window of time series (window_size × features). Encoder LSTM: compress sequence → final hidden state = latent vector. Decoder LSTM: reconstruct sequence từ latent. Train trên normal sequences. Inference: reconstruction error cao → anomaly. Dùng cho: equipment sensor data, network traffic, financial fraud |
| **DBSCAN cho outlier detection** | [Scaler — DBSCAN](https://www.scaler.com/topics/machine-learning/dbscan-for-anomaly-detection/) | DBSCAN = Density-Based Spatial Clustering of Applications with Noise. Density = số points trong ε-neighborhood. Core point: ≥ min_samples neighbors. Border point: < min_samples nhưng reachable từ core. Noise point: không core, không border = outlier. Không cần specify số clusters trước. Points labeled as noise = anomalies. Parameters: eps (neighborhood radius), min_samples |
| **Scikit-learn: IsolationForest, OneClassSVM** | [Sklearn Anomaly](https://scikit-learn.org/stable/modules/outlier_detection.html) | Code: `from sklearn.ensemble import IsolationForest` → `model = IsolationForest(contamination=0.01, random_state=42)` → `predictions = model.fit_predict(X)`. -1 = anomaly, 1 = normal. `model.decision_function(X)` → anomaly scores. `from sklearn.svm import OneClassSVM` → `model = OneClassSVM(kernel='rbf', nu=0.01)`. Autoencoder: `from sklearn.model_selection import train_test_split` → train chỉ trên X_train[normal_mask] |
| **Thực hành: phát hiện gian lận thẻ tín dụng** | [Kaggle Credit](https://www.kaggle.com/datasets/dhanushnarayanan/credit-card-fraud) | Dataset: highly imbalanced (frauds < 1%). Approach 1: Isolation Forest với `contamination` nhỏ (0.001-0.01). Approach 2: Autoencoder trained trên non-fraud transactions. Approach 3: SMOTE + XGBoost classification (supervised). Metric: ROC-AUC, Precision-Recall curve (accuracy không đáng tin với imbalanced data). Xem confusion matrix để đánh giá false positive rate |
| **Thực hành: detect bất thường trong sensor IoT** | Tự tạo dataset + Isolation Forest | Scenario: temperature/pressure sensor trong máy móc công nghiệp. Inject artificial anomalies (sudden spikes, gradual drift). Train Isolation Forest trên normal operating data. Evaluate: % anomalies detected, false alarm rate. Compare: Z-score vs IQR vs Autoencoder |

## 5.3 — Ensemble Methods Nâng Cao

### Mục đích
Ensemble = kết hợp nhiều models để predictions tốt hơn. PHẦN 2 đã học Random Forest, XGBoost, LightGBM. PHẦN 5 đi sâu hơn: Stacking (ensemble of ensembles), CatBoost, Hyperparameter tuning với Optuna — cách đạt top performance trong competitions.

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Boosting — tổng quan** | [Scaler — Boosting](https://www.scaler.com/topics/machine-learning/boosting-algorithm-in-machine-learning/) | Boosting = train weak learners SEQUENTIALLY, mỗi learner tập trung vào errors của learners trước. Khác Bagging (RF): bagging train parallel, boosting train sequential. Boosting thường chính xác hơn nhưng dễ overfit hơn. Types: AdaBoost (weight misclassified samples), Gradient Boosting (optimize loss function), XGBoost/LightGBM (optimized implementations) |
| **Stacking — ensemble of ensembles** | [Scaler — Stacking](https://www.scaler.com/topics/machine-learning/stacking-in-machine-learning/) | Stacking = kết hợp predictions của diverse base models bằng meta-learner. Level 0 (base models): Random Forest, XGBoost, SVM, Logistic Regression — trained on data → output predictions. Level 1 (meta-learner): Logistic Regression hoặc XGBoost — trained on base model predictions. Key: base models phải diverse (khác loại, khác hyperparameters). OOF (Out-of-Fold) predictions dùng để train meta-learner tránh overfit |
| **Blending — biến thể của Stacking** | [Scaler — Blending](https://www.scaler.com/topics/machine-learning/blending-in-machine-learning/) | Blending = simplified version of Stacking. Khác: dùng holdout set thay vì cross-validation để generate base model predictions. Split data: 80% train, 20% holdout. Train base models trên 80% → predict on 20% → meta-learner train trên predictions đó. Đơn giản hơn, less prone to leakage, nhưng sử dụng less data cho base models |
| **Voting Classifier vs Stacking** | [Sklearn Voting](https://scikit-learn.org/stable/modules/ensemble.html#voting-classifier) | Voting: combine predictions bằng majority vote (hard) hoặc average probabilities (soft). Đơn giản nhưng không học optimal combination weights. Stacking: train meta-learner trên base predictions → học optimal combination. Stacking thường better khi base models có complementary strengths. Code: `VotingClassifier([('rf', rf), ('xgb', xgb), ('svc', svc)], voting='soft')` |
| **CatBoost — Boosting cho categorical** | [Scaler — CatBoost](https://www.scaler.com/topics/machine-learning/catboost-algorithm/) | CatBoost = Categorical Boosting, thiết kế đặc biệt cho data có nhiều categorical features. Dùng Target Encoding với Ordered Statistics (ngăn leakage). Xử lý categorical tự động, không cần one-hot encoding. Robust to overfitting trên small datasets. Built-in handling cho missing values. Performance tương đương XGBoost, sometimes better trên high-cardinality categoricals |
| **Extra Trees — Extremely Randomized Trees** | [Scaler — Extra Trees](https://www.scaler.com/topics/machine-learning/extra-trees-classifier/) | Extra Trees = variant của Random Forest. Khác RF: thay vì find optimal split, dùng random splits (chọn random thresholds). Faster training vì không optimize splits. More randomization → more bias nhưng lower variance. Phù hợp khi nhiều noisy features. `ExtraTreesClassifier(n_estimators=500)` |
| **Histogram-based Gradient Boosting** | [Sklearn HistGB](https://scikit-learn.org/stable/modules/ensemble.html#histogram-based-gradient-boosting) | HistGradientBoosting (sklearn) = similar to LightGBM, dùng histogram binning cho continuous features. Faster training khi n_samples large. Native support: missing values, categorical features. API: `HistGradientBoostingClassifier(max_iter=200, learning_rate=0.1)`. Tốt khi muốn sklearn ecosystem mà không cần cài XGBoost/LightGBM riêng |
| **Hyperparameter Tuning — Optuna** | [Optuna](https://optuna.org/) | Optuna = automated hyperparameter optimization framework. Dùng Bayesian Optimization (TPE algorithm) thay vì Grid/Random Search. TPE: builds probabilistic model P(params\|performance) → sample params có high likelihood of being good. Faster than random/grid search, especially for large search spaces. Integration: sklearn, XGBoost, LightGBM, PyTorch. Code: define objective function → `study = optuna.create_study(direction='maximize')` → `study.optimize(objective, n_trials=100)` |
| **Thực hành: so sánh 10 ensemble trên 1 dataset** | Kaggle Titanic / Wine | Train 10+ models: Logistic Regression, KNN, SVM, Decision Tree, Random Forest, AdaBoost, XGBoost, LightGBM, CatBoost, HistGB, Stacking Ensemble. Compare: accuracy, F1, AUC, training time, inference time. Tạo visualization: radar chart comparing models across metrics. Kết luận: model nào best cho dataset này và tại sao |
| **Thực hành: Stacking Classifier cho credit scoring** | [Kaggle Credit](https://www.kaggle.com/datasets/dhanushnarayanan/credit-card-fraud) | Level 0 models: XGBoost, LightGBM, CatBoost, Random Forest, Logistic Regression. Level 1 meta-learner: Logistic Regression (interpretable) hoặc XGBoost. Use 5-fold CV để generate OOF predictions → train meta-learner. Compare: single XGBoost vs Stacking. Improvement thường 1-3% AUC. Interpret: xem coefficients của meta-learner để hiểu contribution của từng base model |

## 5.4 — Probabilistic ML & Bayesian Methods

### Mục đích
Standard ML: output = single prediction (point estimate). Probabilistic ML: output = probability distribution → có thể quantify uncertainty. Quan trọng trong medical, finance, scientific applications — nơi cần biết "chắc chắn bao nhiêu %".

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Probabilistic ML — tại sao cần probability** | [Scaler — Prob ML](https://www.scaler.com/topics/machine-learning/probabilistic-machine-learning/) | Point estimate: model.predict() → single number. Probabilistic: model.predict() → distribution. Ví dụ: dự đoán giá nhà $500K ± $50K (có uncertainty). Uncertainty types: Aleatoric (inherent noise trong data: measurement error) và Epistemic (model uncertainty: cần more data). Probabilistic ML đặc biệt quan trọng khi: small data, high-stakes decisions, out-of-distribution detection |
| **Maximum Likelihood Estimation (MLE)** | [Scaler — MLE](https://www.scaler.com/topics/machine-learning/maximum-likelihood-estimation/) | MLE = tìm parameters θ sao cho P(data \| θ) là maximum. Likelihood = probability của data given parameters. Ví dụ: data = [heads, tails, heads].假设 p = probability of heads. Likelihood L(p) = p × (1-p) × p = p²(1-p). MLE: derivative = 0 → p = 2/3. MLE là nền tảng của hầu hết ML: Logistic Regression, Neural Networks đều optimize likelihood |
| **Bayesian Linear Regression** | [Scaler — Bayesian LR](https://www.scaler.com/topics/machine-learning/bayesian-linear-regression/) | Khác Linear Regression thông thường: thay vì single point estimate cho weights, dùng probability distribution over weights. Prior P(θ): prior belief về weights. Likelihood P(y\|X, θ): how likely data given weights. Posterior P(θ\|X,y) ∝ P(y\|X,θ) × P(θ) (Bayes theorem). Predictive distribution P(y*\|x*, X,y): bao gồm uncertainty về both weights và inherent noise. Posterior thường intractable analytically → dùng approximation (VI, MCMC) |
| **Gaussian Process Regression** | [Scaler — GP](https://www.scaler.com/topics/machine-learning/gaussian-process-in-machine-learning/) | GP = generalization của Gaussian distribution sang infinite dimensions. Prior over functions: f(x) ~ GP(m(x), k(x, x')). m(x) = mean function (thường = 0). k(x, x') = kernel function (RBF, Matern...). Posterior: P(f*\|X*, X, y). Output: mean prediction + confidence interval. Ưu điểm: tự động quantify uncertainty, work well với small data. Nhược: O(n³) complexity → không scale cho large datasets. Dùng khi: scientific experiments, Bayesian optimization |
| **Variational Inference** | [Scaler — VI](https://www.scaler.com/topics/machine-learning/variational-inference/) | VI = approximate posterior distributions tractable bằng optimization. Intractable posterior P(θ\|D) → approximate với q(θ) từ family (Gaussian). Minimize KL(q(θ) \|\| P(θ\|D)) → tìm q gần posterior nhất. ELBO (Evidence Lower BOund) = cách optimize VI. Variational Autoencoder (VAE) = VI applied to deep learning. Faster than MCMC, scale được to large data, nhưng approximate (không exact như MCMC) |
| **EM Algorithm — Expectation Maximization** | [Scaler — EM](https://www.scaler.com/topics/machine-learning/em-algorithm-in-machine-learning/) | EM = iterative algorithm cho latent variable models. E-step: given current parameters, compute expected values của latent variables. M-step: given expected latent values, update parameters to maximize expected likelihood. Repeat until convergence. Dùng khi: data có missing values, mixture models (GMM), hidden states (HMM). Convergence guaranteed to local maximum của likelihood |
| **PyMC — Bayesian modeling** | [PyMC Official](https://www.pymc.io/) | PyMC = thư viện Python cho Bayesian statistical modeling. Cho phép define probabilistic models bằng Python code. Hỗ trợ: MCMC sampling (NUTS, Metropolis-Hastings), Variational Inference. Code pattern: `with pm.Model() as model: slope = pm.Normal('slope', mu=0, sigma=10); obs = pm.Normal('obs', mu=slope*x, sigma=1, observed=y); trace = pm.sample(2000)` → lấy posterior distributions. Dùng cho: Bayesian regression, hierarchical models, mixture models |
| **Thực hành: Bayesian regression cho price prediction** | Kaggle dataset + PyMC | Dataset: house prices với nhiều features. Bayesian Linear Regression với PyMC. Output: posterior distributions của coefficients → uncertainty estimates cho mỗi feature. Compare: frequentist (sklearn LinearRegression) vs Bayesian. Visualization: posterior distributions của weights, credible intervals. Benefit: interpretability cao hơn, có uncertainty |
| **Thực hành: Gaussian Process cho uncertainty estimation** | Scikit-learn GP + housing | Dùng sklearn `GaussianProcessRegressor`. Kernel: RBF + WhiteKernel (noise). Compare predictions: GP mean vs XGBoost point estimate. Visualize: prediction ± 1σ, 2σ confidence intervals. Use case: scientific measurement, drug dosage prediction — nơi cần biết uncertainty |

## 5.5 — Graph Machine Learning

### Mục đích
Graph data = mạng lưới các đối tượng có quan hệ (social network, citation, molecule, transportation). Khác tabular/sequential data: không có fixed structure — nodes có thể có different numbers of neighbors. Graph ML học representations cho nodes, edges, hoặc entire graphs.

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Graph ML — tại sao cần** | [Scaler — Graph ML](https://www.scaler.com/topics/graph-machine-learning/graph-theory-for-ml/) | Graph = Nodes (đỉnh) + Edges (cạnh) + Optional: Node/Edge features. Examples: Social network (users = nodes, friendships = edges), Citation (papers = nodes, citations = edges), Molecule (atoms = nodes, bonds = edges), Transportation (locations = nodes, roads = edges). Traditional ML: each node independent. Graph ML: encode relationships/structure vào features. Rất nhiều real-world data là graphs |
| **Adjacency Matrix & Graph Laplacian** | [Scaler — Adjacency](https://www.scaler.com/topics/graph-machine-learning/graph-representation-learning/) | Adjacency matrix A (n×n): A[i,j] = 1 nếu có edge giữa i và j, else 0. Undirected: symmetric. Directed: asymmetric. Weighted: A[i,j] = weight. Degree matrix D: diagonal, D[i,i] = degree của node i. Graph Laplacian L = D - A. Eigenvectors của L → spectral clustering, graph Fourier transform. Cách biểu diễn graph để compute trên nó |
| **Node2Vec — random walk → embedding** | [Scaler — Node2Vec](https://www.scaler.com/topics/graph-machine-learning/node2vec/) | Node2Vec = map nodes thành vectors (embeddings). Random walks: từ node, simulate random walk length L để get sequence of nodes. Use Word2Vec (Skip-gram) để learn node embeddings từ random walk sequences. p parameter: control return probability (BFS-like vs DFS-like). q parameter: control exploration vs exploitation. Embeddings capture structural equivalence và community structure. Nodes có similar roles → similar embeddings |
| **GNN — Message Passing** | [Scaler — GNN](https://www.scaler.com/topics/graph-machine-learning/graph-neural-network/) | GNN = class of neural networks cho graph data. Message Passing Framework: mỗi iteration (layer): (1) Message: each node gửi thông tin cho neighbors. (2) Aggregate: combine messages từ neighbors (mean, sum, max, attention). (3) Update: update node's hidden state dựa trên aggregated messages. K layers = K-hop neighborhood. Sau K layers, node embeddings chứa thông tin từ K-hop neighbors |
| **GCN — Graph Convolutional Network** | [Scaler — GCN](https://www.scaler.com/topics/graph-machine-learning/gcn-graph-convolutional-network/) | GCN = convolution operation trên graph. Spectral approach: dùng Laplacian eigenvectors. Spatial approach: aggregate neighborhood features. Layer: H^{(l+1)} = σ(D^{-1/2} A D^{-1/2} H^{(l)} W^{(l)}). A: adjacency + self-loops. D^{-1/2} A D^{-1/2}: symmetric normalization. 2 layers GCN → 2-hop neighborhood. Semisupervised node classification: label only a few nodes → propagate labels |
| **GAT — Graph Attention Network** | [Scaler — GAT](https://www.scaler.com/topics/graph-machine-learning/graph-attention-network/) | GAT = thay normalization cố định (như GCN) bằng attention weights. Attention coefficient: α_{ij} = softmax_j(LeakyReLU(a^T [W h_i || W h_j])). Nodes có higher attention weights → contribute more trong aggregation. Multi-head attention: nhiều attention heads, outputs concatenated. GAT học được importance của different neighbors → better than uniform aggregation |
| **GraphSAGE — sampling neighbors** | [Scaler — GraphSAGE](https://www.scaler.com/topics/graph-machine-learning/graphsage/) | GraphSAGE = Scalable Graph Representation Learning. Inductive capability: train on one graph, generalize to new unseen graphs. Neighbor sampling: không aggregate ALL neighbors (expensive), chỉ sample K neighbors. Aggregation: Mean, LSTM, Pooling. Mini-batch training: sample fixed-size neighborhoods → scale được cho large graphs (millions of nodes). Foundation của PinSage (Pinterest recommendations) |
| **PyG — PyTorch Geometric** | [PyG Official](https://pytorch-geometric.readthedocs.io/en/latest/) | PyTorch Geometric = thư viện chuẩn cho GNN trong PyTorch. Cung cấp: Dataset loaders (Planetoid, PPI, Reddit, Amazon), Message Passing API (GCNConv, GATConv, SAGEConv), Utility functions. Code: `from torch_geometric.datasets import KarateClub` → `data = dataset[0]`. `model = GCN(data.num_node_features, hidden_channels=16, num_classes=4)` → `model(data)`. Đây là thư viện chuẩn để implement GNN |
| **Link Prediction — dự đoán kết nối mới** | [Scaler — Link Pred](https://www.scaler.com/topics/graph-machine-learning/link-prediction-in-graphs/) | Task: predict whether edge tồn tại giữa 2 nodes. Applications: friend recommendation (sẽ kết bạn không?), drug interaction (sẽ tương tác không?). Approach: encode nodes → compute similarity (dot product, L2 distance) → score. Có thể dùng node embeddings từ Node2Vec, GCN. Binary classification: positive edges (exist) vs negative edges (don't exist) |
| **Community Detection** | [Scaler — Community](https://www.scaler.com/topics/graph-machine-learning/community-detection-in-graphs/) | Task: phân nhóm nodes thành communities (clusters). Nodes trong cùng community có nhiều edges hơn giữa các communities. Algorithms: Louvain (modularity optimization), Spectral clustering (dùng Laplacian eigenvectors). Modularity: measure quality của partition. Community detection = unsupervised clustering trên graph structure |
| **Thực hành: friend recommendation bằng Node2Vec** | [Stanford Facebook](https://snap.stanford.edu/data/egonets/Facebook.html) + PyG | Dataset: Facebook ego networks (users + friendships). Train Node2Vec embeddings. Link Prediction: given 2 users, predict whether they'll become friends. Evaluate: ROC-AUC, Precision@K. Compare: Node2Vec vs GCN embeddings. Visualization: t-SNE của embeddings, color by ground-truth community |
| **Thực hành: phân loại bài báo bằng GCN** | [CiteSeer Graph](https://pytorch-geometric.readthedocs.io/en/latest/generated/torch_geometric.datasets.Planetoid.html) | CiteSeer: citation network, nodes = papers, edges = citations. Features: bag-of-words. Task: classify papers into 6 categories. Train GCN (2 layers) → semi-supervised (label only 20 nodes per class). Compare: GCN vs GAT vs GraphSAGE. Metrics: accuracy. Visualization: t-SNE của learned node embeddings |

## 5.6 — Reinforcement Learning

### Mục đích
Reinforcement Learning = học từ tương tác với môi trường. Khác Supervised (có labels), khác Unsupervised (tìm patterns): RL = Agent học từ Rewards (phần thưởng) sau mỗi Actions. Cốt lõi của game AI, robotics, autonomous systems, và LLM alignment (RLHF).

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **RL — Agent, Environment, Reward, State, Action** | [Scaler — RL Intro](https://www.scaler.com/topics/reinforcement-learning/what-is-reinforcement-learning/) | Agent = learner/decision maker. Environment = thế giới agent tương tác. State (s) = current situation của environment. Action (a) = agent's choice. Reward (r) = feedback từ environment. Policy π(s) → a = hành động tối ưu theo state. Goal: maximize cumulative reward (return) over time. Temporal credit assignment: hành động ở step trước ảnh hưởng reward hiện tại |
| **MDP — Markov Decision Process** | [Scaler — MDP](https://www.scaler.com/topics/reinforcement-learning/markov-decision-process/) | MDP = mathematical framework cho RL. Defined by (S, A, P, R, γ). S = states. A = actions. P(s'\|s,a) = transition probability. R(s,a) = reward function. γ = discount factor (0-1). Markov property: future depends only on current state, not history. Goal: find optimal policy π* maximizing expected discounted return G₀ = Σ γ^t r_t |
| **Value Function & Bellman Equation** | [Scaler — Bellman](https://www.scaler.com/topics/reinforcement-learning/bellman-equation-in-rl/) | State Value Function V(s) = expected return starting from state s. V(s) = E[G_t \| S_t = s]. Action Value Function Q(s,a) = expected return starting from s, taking action a. Bellman Equation: V(s) = Σ_a π(a\|s) Σ_{s'} P(s'\|s,a) [R(s,a,s') + γ V(s')]. Optimal: V*(s) = max_a Σ_{s'} P(s'\|s,a) [R + γ V*(s')]. Basis của mọi RL algorithms |
| **Q-Learning — table-based** | [Scaler — Q-Learning](https://www.scaler.com/topics/reinforcement-learning/q-learning-in-reinforcement-learning/) | Q-Learning = off-policy TD learning. Q(s,a) = expected return nếu start từ s, do a, sau đó follow optimal policy. Update: Q(s,a) ← Q(s,a) + α [r + γ max_{a'} Q(s',a') - Q(s,a)]. Off-policy: học optimal policy từ exploratory behavior. Không cần model của environment (P transitions). Chỉ work cho discrete, small state/action spaces (Q-table) |
| **Deep Q-Network (DQN)** | [Scaler — DQN](https://www.scaler.com/topics/reinforcement-learning/deep-q-network/) | DQN = Q-Learning với deep neural network thay Q-table. Q(s,a;θ) ≈ optimal Q(s,a). Network input: state. Output: Q-values cho all actions. Two key innovations: (1) Experience Replay: store (s,a,r,s') in buffer, sample random batch → breaks correlation. (2) Target Network: separate target network với weights θ⁻ → stable targets. Loss = (r + γ max Q(s',a';θ⁻) - Q(s,a;θ))² |
| **Experience Replay & Target Network** | [Scaler — DQN Improve](https://www.scaler.com/topics/reinforcement-learning/deep-q-network/) | Experience Replay Buffer: finite-size circular buffer, store transitions. Sample uniformly random batch → decorrelates samples → like supervised learning. Benefits: more efficient data usage (each sample used multiple times), breaks temporal correlation. Target Network: Q-targets = r + γ max_a' Q(s',a';θ⁻). Without it: targets move as network updates → unstable. Update θ⁻ ← θ periodically (every N steps) hoặc soft update (τ × θ + (1-τ) × θ⁻) |
| **Policy Gradient — REINFORCE** | [Scaler — Policy Grad](https://www.scaler.com/topics/reinforcement-learning/policy-gradient-method/) | Policy Gradient: directly optimize policy π(a\|s;θ) bằng gradient ascent. Gradient: ∇_θ J = E[∇_θ log π(a\|s) × G_t]. G_t = return from time t. Monte Carlo estimation: sample trajectories → estimate returns → compute gradient. Advantages: continuous actions, stochastic policies. Disadvantages: high variance, slow convergence. REINFORCE: baseline subtraction để reduce variance |
| **PPO — Proximal Policy Optimization** | [Scaler — PPO](https://www.scaler.com/topics/reinforcement-learning/ppo-proximal-policy-optimization/) | PPO = thuật toán RL phổ biến nhất hiện nay. Clipped surrogate objective: L^{CLIP}(θ) = E[min(r_t(θ) Â_t, clip(r_t(θ), 1-ε, 1+ε) Â_t)]. r_t(θ) = π_θ(a\|s) / π_{θ_old}(a\|s) = probability ratio. Clip prevents too-large policy updates → stable. PPO-Clip: simpler, more robust. PPO-Penalty: KL constraint. Trust Region: PPO ≈ TRPO but simpler to implement |
| **A2C / A3C — Advantage Actor-Critic** | [Scaler — A2C](https://www.scaler.com/topics/reinforcement-learning/actor-critic-method/) | Actor-Critic: combine policy gradient (Actor) + value function (Critic). Actor: π(a\|s;θ) — learns policy. Critic: V(s;w) — estimates value function. Advantage A(s,a) = Q(s,a) - V(s) = r + γV(s') - V(s). Reduces variance socia với pure PG. A2C: synchronous, multiple workers → collect gradients → update. A3C: asynchronous workers → different exploration → more diverse. Gần như được thay thế bởi PPO trong practice |
| **RLHF — RL từ Human Feedback** | [Scaler — RLHF](https://www.scaler.com/topics/generative-ai/reinforcement-learning-from-human-feedback/) | RLHF = cách fine-tune LLMs để align với human preferences. Step 1: SFT (Supervised Fine-tuning) trên demonstration data. Step 2: Reward Model — train neural network predict human preference: given (prompt, response₁, response₂) → predict which is better. Step 3: PPO fine-tune LLM để maximize learned reward model. Output: ChatGPT, Claude, GPT-4 behavior = result của RLHF. Reward hacking là challenge lớn |
| **Gymnasium — môi trường RL chuẩn** | [Gymnasium](https://gymnasium.farama.org/) | Gymnasium (trước OpenAI Gym) = standard API cho RL environments. `env = gym.make('CartPole-v1')` → `observation, info = env.reset()` → `observation, reward, terminated, truncated, info = env.step(action)`. Environments: Classic control (CartPole, Pendulum), Atari (breakout, pong), MuJoCo (robotics), Toy text (gridworld). Đây là tiêu chuẩn để benchmark và reproduce RL research |
| **Stable Baselines3 — RL library** | [SB3 Docs](https://stable-baselines3.readthedocs.io/en/master/) | SB3 = production-ready RL implementations. Wraps Gymnasium + implements popular algorithms. Available: DDPG, TD3, SAC (continuous control), PPO, A2C (policy gradient), Q-Learning variants. Code: `from stable_baselines3 import PPO` → `model = PPO('MlpPolicy', env, verbose=1)` → `model.learn(total_timesteps=100000)` → `model.predict(obs)`. Evaluation, model saving, loading đều built-in |
| **Thực hành: chơi CartPole bằng DQN** | Gymnasium + SB3 | CartPole: pole balanced on cart. State: cart position/velocity, pole angle/angular velocity. Actions: push left/right. Reward: 1 per timestep pole stays up. Task: survive > 475 steps (solved). Implement DQN: replay buffer, target network, epsilon-greedy. Train → plot cumulative reward. Evaluate: average reward over 100 episodes. Compare: DQN vs PPO vs random policy |
| **Thực hành: trading bot bằng PPO** | [Kaggle Stock](https://www.kaggle.com/datasets/jacksoncrow/stock-market-dataset) + SB3 | Environment: custom trading env. State: price history (lookback window). Actions: Buy, Sell, Hold. Reward: profit/loss per trade. PPO: train agent trên historical stock data. Challenges: reward shaping, action space design, overfitting to historical data. Evaluate: Sharpe ratio, max drawdown trên test period. Caveat: past performance ≠ future results |

## 5.7 — Meta-Learning & AutoML

### Mục đích
Meta-Learning = "học cách học" (learning to learn). Thay vì train model từ scratch cho mỗi task mới, meta-learning transfer knowledge về cách học hiệu quả. AutoML = tự động hóa ML pipeline — chọn algorithms, hyperparameters, feature engineering — để non-expert cũng có thể deploy models tốt.

### Chi tiết từng bước

| Topic con | Tài liệu | Mô tả chi tiết |
|---|---|---|
| **Meta-Learning — học cách học** | [Scaler — Meta Learning](https://www.scaler.com/topics/machine-learning/meta-learning-in-machine-learning/) | Standard ML: train on task T → good model. Meta-Learning: train on MANY tasks → good LEARNER/algorithm. Meta-train: tasks T₁, T₂, T₃... → meta-knowledge. Meta-test: new task T_new → adapt quickly. Three approaches: (1) Metric-based (learn similarity: Siamese, Prototypical networks), (2) Model-based (learn with external memory: MANN, Memory-augmented NN), (3) Optimization-based (learn initialization/optimizer: MAML, Reptile) |
| **Few-Shot Learning** | [Scaler — Few Shot](https://www.scaler.com/topics/machine-learning/few-shot-learning-in-machine-learning/) | Few-shot = learn new concept từ rất ít examples (1-shot = 1 example, 5-shot = 5 examples). N-way K-shot classification: N classes, K examples each. Metric-based approach: learn embedding space, compare new examples to class prototypes. Prototypical Networks: encode support set → class prototypes → classify query by nearest prototype. Data augmentation in embedding space → increase support set diversity |
| **Transfer Learning — deep dive** | [Scaler — Transfer Deep](https://www.scaler.com/topics/machine-learning/transfer-learning-in-deep-learning/) | Transfer = knowledge từ source task/domain → improve learning/performance on target task/domain. Fine-tuning = transfer learning + continue training on target data. Domain shift = source và target distributions khác nhau. Feature extraction: freeze backbone, train only classifier head. Fine-tuning: unfreeze last layers (hoặc full network) với small learning rate. Progressive unfreezing: unfreeze gradually để avoid catastrophic forgetting |
| **Multi-Task Learning** | [Scaler — Multi Task](https://www.scaler.com/topics/machine-learning/multi-task-learning/) | Multi-Task = train single model trên multiple related tasks simultaneously. Shared encoder learns representations useful for ALL tasks. Hard parameter sharing: all tasks share hidden layers. Soft parameter sharing: separate networks, distance regularization encourages similarity. MTL benefits: regularization effect (shared representations), data augmentation effect (more training signal), knowledge transfer between tasks. Caveat: task interference, negative transfer |
| **Neural Architecture Search (NAS)** | [Scaler — NAS](https://www.scaler.com/topics/deep-learning/neural-architecture-search/) | NAS = tự động tìm optimal neural network architecture. Search space: candidate operations (conv, pooling), connections. Search strategy: random search, RL (predictor-based), evolutionary, gradient-based (DARTS). Evaluation: train candidate architectures → measure performance (expensive). Efficiency techniques: weight sharing (share weights across candidates), early stopping. NASNet, EfficientNet (NASNet + compound scaling), AutoML-Zero (evolve from scratch) |
| **AutoML — tự động hóa ML pipeline** | [Auto-Sklearn](https://automl.github.io/auto-sklearn/) | AutoML = full pipeline automation: feature preprocessing, model selection, hyperparameter tuning, ensemble. Auto-sklearn: dựa trên scikit-learn. Bayesian optimization cho hyperparameters + meta-learning (warm-start from similar datasets). AutoGluon: AWS AutoML, multi-layer stacking, feature engineering included. H2O AutoML: tree-based models focus. Google Cloud AutoML: hosted, easy to use. Compare: AutoML vs human expert → AutoML often competitive |
| **Optuna — hyperparameter optimization** | [Optuna Official](https://optuna.org/) | Optuna = Bayesian HP tuning framework. TPE (Tree-structured Parzen Estimator): model P(good params \| performance). Sampler: suggest next params based on model. Pruner: early stop unpromising trials. Integrations: sklearn, XGBoost, LightGBM, PyTorch, Keras. Dashboard: visualize search progress. Code: `study = optuna.create_study(direction='maximize')` → `study.optimize(objective, n_trials=100, timeout=600)` → best parameters. Often finds better params than grid/random search |
| **Keras Tuner** | [Keras Tuner](https://keras.io/keras_tuner/) | Keras Tuner = HP tuning cho TensorFlow/Keras. Search spaces: Int, Float, Choice, conditional spaces. Tuners: RandomSearch, Hyperband (successive halving), BayesianOptimization. Hyperband: train all candidates for few epochs → prune bottom half → repeat. Code: `tuner = kt.Hyperband(build_model, objective='val_accuracy', max_epochs=50)` → `tuner.search(X_train, y_train, validation_data=(X_val, y_val))` → `tuner.get_best_hyperparameters()[0]` |
| **Thực hành: Few-shot classification** | Mini-ImageNet + PyTorch | Omniglot (1623 characters, 20 examples each): benchmark few-shot. Mini-ImageNet: subset of ImageNet. N-way K-shot task: sample N classes → K support + query images → classify query. Prototypical Networks: encode → compute class prototypes → classify by nearest prototype. Compare: 1-shot vs 5-shot accuracy. Fine-tune baseline: update embeddings on support set → compare |
| **Thực hành: AutoML cho tabular data** | Auto-sklearn + Kaggle | Dataset: Kaggle tabular competition. Run Auto-sklearn: `automl = AutoMLClassifier(time_left_for_this_task=3600)` → `automl.fit(X_train, y_train)` → `predictions = automl.predict(X_test)`. Compare with manually tuned XGBoost/LightGBM. H2O AutoML: `h2o.automl(...)` → leaderboard of models. Evaluate: feature importance, model diversity, time cost. Insight: when is manual tuning worth it vs AutoML? |
