# Workflow Phân Tích Đầy Đủ — Thị Trường + Lộ Trình Tăng Trưởng

## Mục đích
Phân tích chi phí toàn diện kết hợp tư duy thị trường, so sánh đối thủ, và lộ trình phát triển dài hạn. Dùng khi sinh viên cần trình bày bài toán kinh doanh (BTKD) cho thầy/cô hoặc nhà đầu tư.

## Khi nào dùng
- User cần làm bài toán kinh doanh (BTKD) cho trường
- User cần pitch cho nhà đầu tư hoặc thầy/cô
- User muốn phân tích sâu đối thủ cạnh tranh
- User muốn lộ trình 12–36 tháng đầy đủ

## Điều kiện trước
Cần có:
- Tên sản phẩm AI
- Chi phí cố định + biến đổi (hoặc ước lượng)
- Số user mục tiêu theo giai đoạn
- Mức lương mục tiêu sau ra trường
- Phân khúc khách hàng mục tiêu (sinh viên, doanh nghiệp nhỏ...)

## Các bước

### Bước 1: Phân tích phân khúc thị trường

Trước khi tính chi phí, cần hiểu AI đang bán cho ai:

```
## 🏪 PHÂN TÍCH THỊ TRƯỜNG

### Ai đang mua sản phẩm AI tại Việt Nam 2024–2025?

| Phân khúc | Đặc điểm | Sẵn sàng trả | Ví dụ sản phẩm |
|-----------|---------|--------------|-----------------|
| Sinh viên / Cá nhân | Ngân sách 0–200K/tháng | Rất thấp | Tool miễn phí, freemium |
| Freelancer | 200K–500K/tháng | Thấp–trung bình | Content tool, design tool |
| Startup nhỏ | 500K–2M/tháng | Trung bình | Chatbot, automation |
| Doanh nghiệp nhỏ | 2M–10M/tháng | Cao | CRM AI, workflow AI |
| Doanh nghiệp lớn | 10M+/tháng | Rất cao | Custom AI solution |

### Bạn đang nhắm phân khúc nào?
→ [Phân khúc]: [Lý do chọn]

**Tư duy thị trường:** Nếu bạn bán cho sinh viên nhưng đặt giá doanh nghiệp →
thất bại. Nếu bạn bán cho doanh nghiệp nhưng sản phẩm yếu → thất bại.
Chọn 1 phân khúc, hiểu sâu, mở rộng sau.
```

### Bước 2: So sánh với đối thủ cạnh tranh

Research nhanh (hoặc dùng kiến thức có sẵn):

```
### Đối thủ trực tiếp & giá thị trường

| Đối thủ | Giá gói tháng | Tính năng chính | Điểm mạnh |
|---------|--------------|-----------------|-----------|
| [Tên] | xxx.xxx VND | | |
| [Tên] | xxx.xxx VND | | |
| [Sản phẩm của bạn] | xxx.xxx VND | | |

### Vị thế của bạn:
- [ ] Rẻ hơn đối thủ → Cạnh tranh giá (khó giữ lâu dài)
- [ ] Đắt hơn đối thủ → Cần tính năng vượt trội rõ ràng
- [ ] Giá tương đương → Cần khác biệt hóa khác (UI, hỗ trợ, tốc độ)

**Câu hỏi then chốt:** Tại sao khách hàng chọn BẠN thay vì đối thủ miễn phí?
```

### Bước 3: Bảng chi phí đầy đủ (từ SKILL.md)

Thực hiện Bước 1–6 trong SKILL.md, lấy đầy đủ bảng chi phí.

### Bước 4: Phân tích tỉ giá nâng cao — Break-even theo kịch bản

```
## 📈 PHÂN TÍCH BREAK-EVEN THEO KỊCH BẢN

### Kịch bản 1: Bi quan (worst case)
- User: [x], churn 50%/tháng
- Doanh thu thực: [x] VND/tháng
- Chi phí: [x] VND/tháng
- Kết quả: [Lỗ/Lã] xxx.xxx VND/tháng
- Bạn chịu được bao lâu? → [x] tháng

### Kịch bản 2: Trung bình (realistic)
- User: [x], churn 20%/tháng
- Doanh thu thực: [x] VND/tháng
- Chi phí: [x] VND/tháng
- Kết quả: [Lỗ/Lã] xxx.xxx VND/tháng

### Kịch bản 3: Lạc quan (best case)
- User: [x], churn 10%/tháng
- Doanh thu thực: [x] VND/tháng
- Chi phí: [x] VND/tháng
- Kết quả: [Lỗ/Lã] xxx.xxx VND/tháng

**Tư duy quan trọng:** Sinh viên thường chỉ tính kịch bản tốt → sốc khi thực tế xảy ra.
Luôn chuẩn bị kịch bản bi quan trước.
```

### Bước 5: Lộ trình phát triển đầy đủ — 3 năm

```
## 🗺️ LỘ TRÌNH PHÁT TRIỂN 3 NĂM

### Năm 1: Xây dựng & Validation
| Quý | Mục tiêu user | Doanh thu mục tiêu | Chi phí thực | Ghi chú |
|-----|--------------|-------------------|-------------|---------|
| Q1 | 0–5 | 0 VND | ~200K/tháng | Free tier API |
| Q2 | 5–20 | 200K–1M | ~500K/tháng | Tính phí nhẹ |
| Q3 | 20–50 | 1M–3M | ~1M/tháng | Tối ưu chi phí |
| Q4 | 50–100 | 3M–6M | ~2M/tháng | Có lãi? |

### Năm 2: Tăng trưởng
| Quý | Mục tiêu user | Doanh thu mục tiêu | Chi phí thực | Ghi chú |
|-----|--------------|-------------------|-------------|---------|
| Q1–Q2 | 100–300 | 6M–18M | ~5M/tháng | Đầu tư marketing |
| Q3–Q4 | 300–500 | 18M–30M | ~8M/tháng | Tuyển support |

### Năm 3: Ổn định & Scale
| Quý | Mục tiêu user | Doanh thu mục tiêu | Chi phí thực | Ghi chú |
|-----|--------------|-------------------|-------------|---------|
| Q1–Q2 | 500–1.000 | 30M–60M | ~15M/tháng | Đội ngũ 2–3 người |
| Q3–Q4 | 1.000+ | 60M+ | ~20M/tháng | Tự động hóa |

**Cột mốc quan trọng:**
- ✅ Month 6: Có ít nhất 20 user trả phí → sản phẩm có giá trị
- ✅ Month 12: Hòa vốn运营 → nếu chưa → đánh giá lại chiến lược
- ✅ Month 24: Doanh thu ≥ lương mục tiêu → có thể toàn thời gian
- ✅ Month 36: Scale hoặc pivot → dựa trên số liệu thực
```

### Bước 6: Thiết kế giới hạn tính năng theo gói

Đây là phần quan trọng nhất — giới hạn phải dựa trên chi phí thực tế:

```
## 📦 THIẾT KẾ GIỚI HẠN TÍNH NĂNG THEO CHI PHÍ

### Bước 6a: Tính chi phí/1 request (hoặc/1 token)

Dựa vào model đang dùng và dữ liệu đã thu thập:

Model: [Tên model]
Input tokens trung bình: [X] tokens
Output tokens trung bình: [Y] tokens
Chi phí/1 request = (X × giá input) + (Y × giá output)

### Bước 6b: Tính giới hạn mỗi gói

Công thức: Giới hạn = (Giá gói ÷ Tỉ giá USD/VND) ÷ Chi phí/1 request

| Gói | Giá VND | Giá USD | Chi phí/1 request | Giới hạn (số lần) |
|-----|---------|---------|------------------|-------------------|
| Starter (tuần) | xxK | $0.xxx | $0.00x | ≈ X lần |
| Basic (tháng) | xxK | $0.xx | $0.00x | ≈ X lần |
| Pro (tháng) | xxK | $0.xx | $0.00x | ≈ X lần |
| Enterprise | xxxK | $X.xx | $0.00x | Soft Unlimited |

### Bước 6c: Chọn mô hình cắt tính năng

Áp dụng ít nhất 2 trong 5 mô hình sau:

| Mô hình | Cách cắt | Phù hợp với |
|---------|---------|------------|
| 1. Request quota | Số lần gọi giới hạn | Mọi sản phẩm |
| 2. Feature gating | Tính năng cao cấp chỉ gói Pro+ | Sản phẩm có nhiều tính năng |
| 3. Model tiering | Gói rẻ = model rẻ hơn | Khi có nhiều LLM option |
| 4. Quality/speed | Gói cao = xử lý nhanh hơn | App real-time |
| 5. Support level | Gói cao = hỗ trợ tốt hơn | B2B SaaS |

### Bước 6d: Bảng giới hạn chi tiết

| Tính năng | Starter | Basic | Pro | Enterprise |
|-----------|---------|-------|-----|------------|
| [Tính năng A] | ✅ | ✅ | ✅ | ✅ |
| [Tính năng B] | ✅ | ✅ | ✅ | ✅ |
| [Tính năng C] | ❌ | ✅ | ✅ | ✅ |
| [Tính năng D] | ❌ | ❌ | ✅ | ✅ |
| [Tính năng E] | ❌ | ❌ | ❌ | ✅ |
| Giới hạn requests | X/tuần | X/tháng | X/tháng | Soft cap |
| Giới hạn tokens | XK tokens | XK tokens | X00K tokens | Unlimited |
| Support | ❌ | Email | Priority | Live + SLA |
| History | ❌ | 7 ngày | 90 ngày | 1 năm |
| Export | ❌ | ❌ | ✅ | ✅ |

> ⚠️ **Lưu ý quan trọng:** Starter BẮT BUỘC có ít nhất 1 tính năng bị cắt để user thấy giá trị khi upgrade. Không bán "full tính năng nhưng giới hạn 1 request/tuần" → không ai mua.
```

### Bước 7: Metrics quan trọng — Đo lường sức khỏe doanh nghiệp

```
## 📡 METRICS THEO DÕI HÀNG THÁNG

| Metric | Công thức | Mục tiêu tháng 1 | Mục tiêu tháng 6 |
|--------|----------|-------------------|-----------------|
| MRR (Doanh thu tháng) | User × Giá gói | 500K | 5M |
| Churn Rate | User rời / Tổng user | <10% | <5% |
| CAC | Chi phí marketing / User mới | — | <100K |
| LTV | ARPU × 1/Churn Rate | — | >500K |
| LTV:CAC | LTV / CAC | — | ≥3:1 |

> 💡 **LTV:CAC < 3:1** = Doanh nghiệp đang đốt tiền. Mỗi khách hàng mới tốn nhiều hơn 1/3 giá trị họ mang lại → KHÔNG bền vững.
```
